# 体系C：四步定量筛股策略

## 目标

新增独立选股策略「四步定量筛股模型」，输出 0-100 总分，>60 分 BUY，<40 分 SELL，中间 WATCH。

## 策略定位

- 与体系A（情绪题材+缠论）、体系B（九大经典策略）平级
- 通过 `StrategyFactory` 注册，名称 `four_step`
- 独立选股策略：输入候选股票池，输出做多信号

## 权重分配

| 步骤 | 权重 | 核心逻辑 |
|------|------|---------|
| 第一步：估值排雷 | 25% | 低估值 + 硬财务 |
| 第二步：筹码大清洗 | 25% | 换手率 + 缩量验证（调整版） |
| 第三步：技术面砸透 | 20% | 跌幅 + BIAS + 无量阴跌 |
| 第四步：资金点火 | 30% | 主力大单净流入 + 右侧突破 |

## 第一步：估值排雷（25分）

**条件A - 低估值（核心，达标即满分100%×25分=25分）**
- PE-TTM 或 PB 处于近5年历史分位的 15% 以下
- 计算方式：获取历史PE/PB序列，计算当前值分位

**条件B - 硬财务（辅助，梯度赋分0-100%）**
- 近3年扣非净利润均为正 → 100%
- 近3年中有2年扣非净利润为正 → 60%
- 近3年中有1年扣非净利润为正 → 30%
- 经营现金流近3年中有2年为正 → 加10%（封顶110%）

**评分**：条件A满足给25分，条件B按梯度赋分×25分

## 第二步：筹码大清洗（25分）[调整版]

**条件A - 换手率不低（核心，达标即满分100%×25分=25分）**
- 5日平均换手率 ≥ 1.5%
- 计算：sum(每日成交量/总股本) / 5

**条件B - 缩量验证（辅助，梯度赋分0-100%）**
- 当前成交量 < 20日均量50% → 100%
- 当前成交量 < 20日均量60% → 70%
- 当前成交量 < 20日均量70% → 40%
- 换手率较30日前下降 > 30% → 额外加分20%（封顶120%）

**评分**：条件A满足给25分，条件B按梯度赋分×25分

## 第三步：技术面砸透（20分）

**三个条件均需满足（AND关系），每条件达标即满分，按权重分配**

| 条件 | 阈值 | 权重 |
|------|------|------|
| 空间急跌 | 从近半年最高点跌幅 > 35% | 8分 |
| 偏离度高 | BIAS(10) < -8% | 6分 |
| 无量阴跌 | 买入前3天成交量 < 20日均量50% | 6分 |

计算 BIAS(10) = (当前收盘价 - 10日均线) / 10日均线 × 100%

## 第四步：资金点火（30分）

**条件A - 大单持续进场（核心，梯度赋分）**
- 连续3日主力大单净流入 > 0 → 60%（满分18分中的60%）
- 连续4日主力大单净流入 > 0 → 80%
- 连续5日主力大单净流入 > 0 → 100%

主力大单 = super_big_order + big_order（来自 fund_flow）

**条件B - 右侧突破（额外加分）**
- 某天收盘价放量突破5日均线 → +10分（加分制，不受权重限制）

**评分**：条件A按梯度赋分×18分 + 条件B固定加10分，最高30分

## 评分逻辑

### 总分计算

```
总分 = Step1得分 + Step2得分 + Step3得分 + Step4得分
```

### 信号输出

| 总分区间 | 信号 | Confidence |
|----------|------|------------|
| > 60 | BUY | min(0.3 + 总分/100×0.65, 0.95) |
| 40-60 | WATCH | 0.4 + (总分-40)/20×0.3 |
| < 40 | SELL | max(0.95 - 总分/100×0.65, 0.3) |

### 数据不足处理

- 数据缺失时，该步得 0 分，最终输出 WATCH + 置信度 0.3
- 不使用假数据填充

## 数据来源

| 数据 | 来源 |
|------|------|
| PE/PB分位 | TechnicalCalculator 计算（基于历史K线） |
| 扣非净利润 | AkShare stock_financial_analysis_indicator |
| 经营现金流 | AkShare stock_financial_analysis_indicator |
| 换手率/成交量 | K线数据计算 |
| 主力大单净流入 | AkShare stock_individual_fund_flow 或 Bying get_fund_flow |

## 错误处理

- 数据不存在：该步骤得 0 分
- 计算异常：捕获并返回 WATCH，log 警告
- 网络错误：触发数据层重试机制
