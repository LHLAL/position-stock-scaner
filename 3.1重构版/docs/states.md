# 4 态规范 · Stock Scanner 3.1

**版本**：v1.1 · 2026-06-14
**作用**：每个组件/卡片在 4 种状态下的视觉与行为规范。
**为什么需要**：v1.0 没有任何态的规范 —— Codex [P1] 指出"AI 流式中断"、"未知代码"、"SSE 断流"、"市场休市"四态缺失。一页规范能省 1-2 天返工。

---

## 总览

| 状态 | 触发 | 视觉元素 | 持续 | 可恢复 |
|---|---|---|---|---|
| **loading** | 数据请求中 | spinner / 进度条 / 骨架屏 | < 30s | 是 |
| **empty** | 数据为空（合法） | 居中图标 + 文案 + CTA | 持续 | 否（用户操作改变） |
| **error** | 失败（异常） | 错误码 + 描述 + 重试按钮 | 持续 | 是（重试） |
| **stale** | 数据陈旧（非交易时段/重连中） | 黄条 + 时间戳 + 刷新 | 持续 | 是（自动/手动） |

**设计原则**：
- 4 态独立组件，不互相覆盖（不要"loading 中又显示旧数据"）
- 全部状态在**当前卡片内**展示，不弹 modal
- 都有**键盘可达**的恢复路径（Tab 可达重试按钮）
- **WCAG AA** 对比度：所有态的背景色 + 文字 ≥ 4.5:1

## 一、Loading（加载中）

### 1.1 触发条件

- 初次进入页面（sidebar / 工具栏 / 卡片全部 loading）
- 点击"重新分析"
- 切到新股票（SSE 重建）
- 拖动 K 线周期

### 1.2 视觉规范

**骨架屏优先于 spinner**（避免"全屏旋转木马"）：

```
┌─ 卡片 (loading) ─────────────────────────────┐
│ ████████  (骨架条 80%)                         │
│                                              │
│ ████████████  (骨架条 60%)                     │
│                                              │
│ ████████████████████  (骨架条 90%)             │
└──────────────────────────────────────────────┘
```

- 骨架色：`var(--color-bg-2)`，用 1px 渐变动画 → **v1 取消闪烁动画**，纯静态
- 高度匹配实际内容高度（避免布局抖动 / CLS）
- 表格 loading：渲染前 5 行骨架，虚拟滚动延迟到 200ms

### 1.3 AI 流式输出（特例）

- **不做**整卡 loading —— 卡片始终显示，**逐字**追加文本
- 顶部进度条（薄 2px）显示"思考中 → 写作中 → 完成"三段
- 完成后进度条 1s 内淡出

### 1.4 长任务

- > 5s：显示"预计还需 X 秒"（基于历史均值）
- > 30s：提供"取消"按钮（不阻塞 UI）
- 后端超时（120s）：自动转 error 态

## 二、Empty（空状态）

### 2.1 触发条件（合法空，非错误）

| 场景 | Empty 文案 | CTA |
|---|---|---|
| 首次访问，自选股为空 | "还没有自选股" | "+ 添加第一只" → 弹搜索 |
| 搜索结果 0 匹配 | "没找到 '{query}'" | "试试 {suggestion}" |
| 题材股票列表为空 | "该题材暂无标的" | "切换题材" |
| 新闻为空（非交易时段） | "暂无新闻" | (无) |
| K 线无数据（新上市） | "上市不足 30 日，K 线数据有限" | "切换到周线" | 30 日是经验阈值：A股新股首日有数据，30 个交易日后均线/BBI 等指标才有意义。可调到 20-60 日。 |
| 财务表为空（季节性披露） | "财报尚未披露" | "查看历史财报" |

### 2.2 视觉规范

```
┌─ 卡片 (empty) ────────────────────────────────┐
│                                              │
│              ◇                               │  ← 静态 SVG 插图（无动画）
│           （灰白线条）                          │
│                                              │
│         还没有自选股                            │  ← 文案 · text-1 · 14px
│       添加关注的股票，方便随时查看                │  ← 描述 · text-2 · 13px
│                                              │
│         [ + 添加第一只 ]                        │  ← 主按钮 · accent-1
│                                              │
└──────────────────────────────────────────────┘
```

- 插图用**静态线条**（非 emoji），保持专业感
- 居中布局，距上 32px / 距下 32px
- 至少有一个 CTA（或显式"无操作"）

### 2.3 不允许的 empty 态

- 单纯显示 "No data" 文字（不够友好）
- 用 emoji 表达"空"（专业感不足）
- 没有任何引导（用户不知道怎么恢复）

## 三、Error（错误状态）

### 3.1 触发条件（异常，需用户感知）

| 场景 | 错误码 | 文案 | 恢复路径 |
|---|---|---|---|
| 未知股票代码 | `E_UNKNOWN_CODE` | "无法识别 {code}" | "搜索支持的代码" |
| 数据源全部失败 | `E_DATA_SOURCE_DOWN` | "所有数据源不可用" | "30s 后自动重试" + 手动"立即重试" |
| SSE 重连失败 | `E_SSE_DISCONNECTED` | "实时连接已断开" | "重新连接" |
| AI 调用失败 | `E_AI_UPSTREAM` | "AI 服务暂时不可用" | "使用规则分析降级" + 手动重试 |
| AI 超时 | `E_AI_TIMEOUT` | "AI 响应超时（>120s）" | "重试" |
| 财务数据缺失 | `E_FUNDAMENTAL_MISSING` | "暂未获取到财报" | "查看技术面" |
| 网络断开 | `E_NETWORK` | "网络连接中断" | "等待恢复" |
| 交易时段内意外闭市 | `E_MARKET_HALTED` | "市场数据暂停（临时停牌/突发休市）" | "查看公告" + "5min 后自动重试" |

### 3.2 视觉规范

```
┌─ 卡片 (error) ────────────────────────────────┐
│ ⚠  数据源全部失败                          [×] │  ← 标题 · accent-3
│   所有数据源（AkShare / Yahoo / 腾讯）均不可用   │  ← 描述 · text-2
│                                              │
│   错误码: E_DATA_SOURCE_DOWN                   │  ← 灰色 · 10px mono
│   最后成功: 14:23:55                          │
│                                              │
│         [ 立即重试 ]  [ 查看日志 ]              │  ← 主按钮 + 次按钮
└──────────────────────────────────────────────┘
```

- 背景：`var(--state-error-bg)`，边框：`var(--color-accent-3)` 1px 左 border
- 错误码必须显示（用户截图反馈时一眼能看到）
- 时间戳告诉用户"是不是很久没工作了"
- **重试按钮永远是主按钮**（不要让用户去找）
- 多次重试失败 → 自动降级路径（如 AI 失败 → 规则分析）

### 3.3 错误聚合（避免淹没）

- 同一时间只显示 1 个错误卡（其他错误进 toast 通知）
- 顶部状态条显示"⚠ N 个错误"汇总
- 点击"查看日志" → 展开底部 log 区

### 3.4 全局错误兜底

```js
window.addEventListener('error', e => {
  showToast({ type: 'error', text: '页面错误，已记录' });
  // Sentry.report(e)  // 后续接入
});
window.addEventListener('unhandledrejection', e => {
  showToast({ type: 'error', text: '操作失败：' + e.reason });
});
```

## 四、Stale（数据陈旧）

### 4.1 触发条件

| 场景 | 检测 | 提示 |
|---|---|---|
| A 股非交易时段 | `now.hour < 9:30 or > 15:00` | "数据更新于 {last_success}，下次刷新 {next_open}" |
| SSE 重连中 | `connectionState = RECONNECTING` | "重连中… (attempt N)" 黄条 + spinner |
| 数据源部分降级 | `degraded_sources: ['akshare']` | "AkShare 不可用，已切到 Yahoo（数据精度可能下降）" |
| 用户切到 tab 超过 5 分钟 | `last_focus > 5min` | 回前台时 toast "数据可能已过期，刷新？" |
| WebSocket/SSE 半开 | `heartbeat_lost > 30s` | "连接可能已断开，刷新？" |

### 4.2 视觉规范

**顶部条（推荐）**：

```
┌──────────────────────────────────────────────┐
│ ⚠  数据更新于 2 小时前 · 非交易时段   [ 刷新 ] │  ← 黄条 · bg-stale
└──────────────────────────────────────────────┘
```

**局部 stale**（仅某张卡）：

```
┌─ 资金流向 (stale) ─────────────────────────────┐
│ ⚠  14:23 后无更新                              │  ← 角标 · 右上
│                                              │
│  主力  +2.3 亿  ████████░░                    │  ← 数据照常显示（不灰掉）
│  ...                                          │
└──────────────────────────────────────────────┘
```

- 角标 / 顶条都用 `var(--color-accent-4)` 黄
- **数据本身不灰化** —— 让用户继续看（避免"看起来坏掉"）
- 自动重连时角标变 spinner，完成后角标淡出

### 4.3 自动刷新策略

| 数据类型 | 刷新间隔 | 触发 |
|---|---|---|
| 实时报价（侧边栏） | 5s | 交易时段 |
| K 线 / 成交量 | 30s | 交易时段 |
| 新闻流 | 60s | 全部时段 |
| 财务数据 | 24h | 仅开市前 |
| AI 分析 | 用户触发 | 不自动 |

**A 股交易时段判定**（修正版，避免伪代码错）：

```js
function isTradingHours(now = new Date()) {
  // 周末直接返回 false
  if (now.getDay() === 0 || now.getDay() === 6) return false;
  const h = now.getHours();
  const m = now.getMinutes();
  const t = h * 60 + m;  // 当前时间转分钟数
  // 早盘 09:30-11:30，午盘 13:00-15:00
  return (t >= 9*60+30 && t < 11*60+30) || (t >= 13*60 && t < 15*60);
}
```

**节假日**：A 股节假日表由后端维护（每年发布），前端不维护。前端 fallback：交易时段内但 5 分钟无更新 → 视为异常闭市（临时停牌/突发休市），按 `E_MARKET_HALTED` 错误码处理（见 §3.1）。

非交易时段全部暂停实时刷新（避免无效请求）。

## 六、键盘可达性（WCAG 2.1）

- **Tab 顺序**：内容 → 状态卡 → 重试/刷新按钮
- **Enter 触发**：默认按钮 = 重试 / CTA
- **Esc 关闭**：toast / modal
- **焦点环**：`focus-visible` 全局 `--focus-ring-width` outline
- **屏幕阅读器**：
  - loading → `aria-live="polite"`，`aria-busy="true"`
  - error → `role="alert"`，`aria-describedby` 指向错误码
  - stale → `aria-live="polite"`，文字描述时间

---

## 七、不允许的反模式

| 反模式 | 替代方案 |
|---|---|
| 整页 spinner 旋转 | 骨架屏 + 当前可看的内容保留 |
| "出错了" 不带错误码 | 必须含 `E_XXX_CODE` |
| 模态弹窗阻断操作 | 卡片内嵌 + toast 通知 |
| 灰化整个卡片表示 stale | 数据照常显示 + 角标提示 |
| 永久 loading（>60s 无响应） | 30s 后转 error + 自动重试 |
| 同一错误反复 toast | 错误聚合到状态条 |
