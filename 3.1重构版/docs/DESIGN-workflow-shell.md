## 一、为什么是这个方向

### 1.1 现状盘点

看了 `templates/index.html`（137KB，**2638 行**内联 JS —— 初次盘点时按空行过滤估成 1664，实际是它的 1.6 倍）和 `3.1重构版/server.log`，当前页面暴露的问题：

| 问题 | 现象 | 影响 |
|---|---|---|
| 横向空间浪费 50% | 股票分析 Tab 主内容只占左半屏 | 屏幕大反而看着空 |
| 数据可视化零 | 全是文字 + 表格 | 看不到 K 线、信号、评分 |
| 配色单调 | 1 种蓝 accent | 没有"金融专业感" |
| 操作按钮溢出 | 持仓巡检"分析/报告/删除"挤到卡片外 | 直接不能用 |
| 实时日志出现两次 | 分析 Tab 上下各一份 | 结构混乱 |
| 表格被中文挤崩 | "人民网"竖排 | 信息密度为负 |

### 1.2 三个被接受的 Premise

```
P1  信息架构按"工作流"组织，不是按"功能"组织
   用户三类场景并存（深度研究 / 多股扫盘 / 持仓监控）
   → 单页面要能服务三类，而不是切 Tab 找功能

P2  持久化侧栏 > 顶部 Tab
   4+ 个目的地 + 上下文信息（自选股、警报、模型状态）
   → 左侧固定栏，顶部留给搜索和全局状态

P3  视觉专业感 = 数据密度
   Bloomberg / TradingView / 同花顺 iFinD 都走这条
   → 不是极简风，是把像素当房租收

P4  图表是必须的
   K 线、信号时序、评分雷达、盈亏曲线，没有等于不存在
```

### 1.3 三个备选 + 决策

| 方案 | 一句话 | 优势 | 代价 | 结论 |
|---|---|---|---|---|
| **A. Workflow Shell** | 左侧自选股 + 三栏主面板 + 底部日志 | 三类场景一屏到位；最专业；可扩展性好 | 改动量大（4-5 天） | **采用** |
| B. Tabbed Dashboard | 保留 4 Tab，重做内部 | 改动小，2 天出活 | "三类并存"体验弱；专业感差 1 档 | 否决 |
| C. Terminal IDE | 可拖拽多面板 + ⌘K 命令面板 | 最强大 | 上手门槛高；2-3 周 | 否决 |

## 三、左侧栏（自选股 + 导航）

### 3.1 结构

```
┌─ 侧边栏 280px ─────────────────┐
│  🔍 搜索代码/名称                │  ← 顶置，可搜代码/名称/拼音
│  [+ 新建分组 ▾]                  │
│                                  │
│  ▼ 持仓 (3)                      │  ← 默认展开
│    600519  贵州茅台   ▲2.3%  ¥1820│
│    603000  人民网     ▼-1.2% ¥19.0│
│                                  │
│  ▶ 自选 (8)                      │  ← 折叠
│    TSLA   特斯拉     ▼-2.1% $248 │
│    ...                           │
│                                  │
│  ▶ 关注板块 (4)                  │  ← 折叠
│    AI / 半导体 / 新能源 / 医药    │
│                                  │
│  ── 底部固定 ──                  │
│  📊 扫盘  🔔 警报  📜 报告  ⚙ 设置│  ← 二级导航，用图标 + tooltip
└──────────────────────────────────┘
```

### 3.2 行为

- 每一行：代码 名称 涨跌幅（着色） 价（等宽）
- 点击：切换主内容区到该股票
- 右键：弹出菜单（加入分组 / 移除 / 设警报 / 复制）
- 拖拽：在分组之间移动
- 双击代码 / 名称：直接进入"代码编辑"模式（v3.1 没有这个能力，先空着）
- 整行 hover：BG-2 高亮 + 显示 ⭐ 收藏
- 涨跌幅 ≥ 5% 或 ≤ -5%：左侧 2px 强调条（绿/红）

## 七、关键交互细节

### 7.1 ⌘K 命令面板

按 `⌘K` / `Ctrl+K` 弹出 overlay，中心化快速操作：

```
┌─ 命令面板 (覆盖层) ──────────────────┐
│  🔍 输入命令 / 代码 / 操作...           │
│  ─────────────────────────             │
│  → 跳到 600519                          │
│  → 切换到扫盘                           │
│  → 分析 600519                          │
│  → 加入自选 600519                     │
│  → 重新分析当前股票                      │
│  → 切换主题 深/浅                       │
│  ...                                    │
└────────────────────────────────────────┘
```

### 7.2 拖拽分割条

主内容区三栏可拖拽：
- 默认 4 : 3 : 3
- 用户拖动后保存到 localStorage
- 双击分割条 → 恢复默认

### 7.3 实时反馈

- SSE 状态：状态条**左侧色块**（绿=连 / 黄=重连中 / 红=断）
- 数据更新：自选股行的**数字**用 cross-fade（不是闪烁），避免视觉疲劳
- AI 输出：流式**逐字**打字，每段 chunk 边界有 0.1s 渐变
- 错误：右上角**浮窗通知**（toast），不弹 modal

### 7.4 响应式降级（次要优先级）

```
1440px+   三栏 + 侧边栏  （默认）
1024-1440 三栏紧凑（40/30/30 → 36/30/34），侧边栏缩到 240px
768-1024  两栏（合并右侧上下文为抽屉），侧边栏 64px 仅图标
< 768     单栏 + 底部 Tab（占位，先保证不崩）
```

### 7.5 性能预算

| 指标 | 目标 |
|---|---|
| 首屏 LCP | < 1.5s（本地） |
| TTI | < 2.0s |
| K 线图首帧 | < 800ms（30 根 K 线数据） |
| 表格 5000 行滚动 | 60fps |
| 内存占用 | < 250MB（10 分钟连续使用） |

### 8.3 引入依赖

```json
{
  "marked": "^12.0",              // 流式 Markdown
```

无前端框架（**不引入 React/Vue**）—— 项目体量不值得。
所有 JS 模块用 ES Modules 原生支持。
